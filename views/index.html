Tecnologias utilizadas:

  • Express.JS

    O Express.js é um framework web altamente popular para Node.js, concebido para simplificar consideravelmente o processo de desenvolvimento de aplicativos web e APIs. Ele funciona como uma camada intermediária entre o servidor e o cliente, proporcionando aos desenvolvedores a capacidade de criar rapidamente rotas, tratar solicitações HTTP, gerenciar middleware e muito mais. Em termos resumidos, o Express.js é empregado nas seguintes áreas:

    • Roteamento: Facilita a criação de rotas que definem como as solicitações HTTP são manipuladas e processadas.
    • Middleware: Permite a implementação de middleware, que são funções intermediárias que podem ser usadas para executar ações em cada solicitação antes que ela alcance o manipulador final.
    • Manipulação de Requisições e Respostas: Torna simples a manipulação de objetos de solicitação (request) e resposta (response) para processamento personalizado.
    • Gerenciamento de Recursos Estáticos: Oferece facilidades para servir recursos estáticos, como arquivos CSS, JavaScript e imagens.
    • Integração com Outras Bibliotecas: Pode ser integrado facilmente com outras bibliotecas e pacotes para estender as capacidades da aplicação.

    No nosso trabalho, o Express.js é empregado em diversos arquivos-chave:

    • Arquivo env: Utilizado para criar o servidor web e gerenciar solicitações HTTP.
    • userController.js: Responsável por definir rotas e tratar solicitações HTTP relacionadas a operações de usuário.
    • routes.js: Utilizado para construir uma API web, criando um conjunto de rotas (endpoints) para lidar com operações relacionadas a usuários em um aplicativo.
    • server.js: Realiza a configuração básica de um servidor web, preparando o ambiente para o uso do Express.js.
  • PostgreSQL

    O PostgreSQL, frequentemente abreviado como "Postgres," é um sistema de gerenciamento de banco de dados relacional de código aberto altamente avançado e amplamente utilizado. Ele é conhecido por sua confiabilidade, desempenho e conjunto abrangente de recursos. O PostgreSQL é uma escolha popular para aplicativos devido à sua robustez. No nosso trabalho, o utilizamos nos seguintes contextos:

    Arquivo env:

    • Declarando as configurações essenciais para a conexão com o banco de dados, incluindo o nome do banco de dados, o nome de usuário e a senha necessários para acessá-lo.
    • Especificando o endereço IP do host onde o banco de dados está hospedado.
    • Definindo a porta que o banco de dados utiliza para aceitar conexões.
    • Configurando o ambiente de execução da aplicação para interagir com o banco de dados.

    userController.js:

    Usado para implementar operações de banco de dados relacionadas a usuários, como criação, leitura, atualização e exclusão de registros.

  • Sequelize CLI

    O Sequelize CLI é uma ferramenta de linha de comando amplamente utilizada em conjunto com o Sequelize, que é um ORM (Object-Relational Mapping) para bancos de dados SQL em JavaScript. Essa combinação é especialmente popular no desenvolvimento de aplicativos Node.js, pois simplifica muitas tarefas relacionadas ao gerenciamento de bancos de dados e modelos.

    O Sequelize CLI desempenha um papel crucial em várias etapas do desenvolvimento de aplicativos, tornando o processo mais eficiente. Suas principais funções incluem:

    • Geração de Modelos: Facilita a criação de modelos de dados que representam tabelas no banco de dados, simplificando o processo de definição de estruturas de dados.
    • Geração de Migrações: Permite criar migrações que descrevem as alterações na estrutura do banco de dados ao longo do tempo, tornando o versionamento do esquema mais gerenciável.
    • Configuração Simplificada: Ajuda os desenvolvedores a configurar facilmente a conexão com o banco de dados, fornecendo um meio conveniente de gerenciar as informações de configuração.
    • Seed de Dados: Facilita a inserção inicial de dados no banco de dados para fins de desenvolvimento e teste.
    • Manutenção do Banco de Dados: Simplifica a manutenção e atualização do banco de dados, tornando mais fácil realizar alterações na estrutura e nos dados.

    No nosso trabalho nos usamos o Sequelize no:

    • config.js: Um arquivo de configuração que armazena as configurações necessárias para a aplicação Node.js interagir com o banco de dados usando o Sequelize.
    • connection.js: Usado para estabelecer uma conexão autenticada com o banco de dados, com base nas informações de configuração fornecidas.
    • userController.js: Utilizado para criar, ler, atualizar e excluir registros no banco de dados, facilitando as operações CRUD (Create, Read, Update, Delete).
    • Arquivos de Migração: São usados para descrever e aplicar modificações na estrutura da tabela do banco de dados, permitindo uma evolução controlada do esquema.
    • Arquivos de Modelo: Lidam com operações de banco de dados SQL relacionadas à recuperação, manipulação e persistência de dados com base nos modelos definidos.

    • '/user' (insert)
    • '/users' (select)
    • '/user/:id' (delete)
    • '/user/:id' (update)
